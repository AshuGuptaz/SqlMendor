Honest by design

⚠ What's real vs. what needs your M1

Everything here that could run truthfully did: the database, a 1,414-pair dataset where every query executes, 56 tests at 79% coverage, and the full self-healing loop — including a real repair (a query referencing a nonexistent column is caught by the database and corrected on retry). The baseline execution accuracy (16%) is the heuristic generator's real score, which is exactly the gap fine-tuning closes.

The fine-tuned model's accuracy and loss are ⟳ run placeholders — MLX trains on Apple Silicon, and this was assembled on Linux, so those run on your M1. The loop guarantees runnable SQL or an abstention; execution accuracy measures correctness. This console is live when the API is up, demo otherwise.
